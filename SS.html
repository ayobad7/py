Buttons & Reminder Chips

Reference sheet for Storage, Borrowed, and Lent. Tweak :root tokens to match brand.

Section Header Actions (“View All”, “Add Item”)

#View All #Add Item
Use a neutral chip for View All so it doesn’t compete with per-item actions. Keep Add Item filled (primary) to make it easy to discover. ✔️ Your current style is appropriate; no change required.

Button Hierarchy (Global)

Primary = most important action on a surface (e.g., Return, Mark Returned). Secondary = useful alternative (e.g., Extend, Remind). Tertiary = light actions (Share, Favorite). Danger = destructive / irreversible (Delete).

Reminder / Due Chips (Borrowed & Lent)

3 days left Due today Overdue 2 days
Borrowed phrasing: “X days left”, “Due today”, “Overdue N days”. Colors: Neutral (far), Warning (soon), Overdue (past due). Pastel tones keep it noticeable without being aggressive.
Due in 3 days Return today Overdue 2 days
Lent phrasing: “Due in X days”, “Return today”, “Overdue N days”. Different wording clarifies that the perspective is the lender’s.

Mark Returned should be Primary (filled accent) — it’s the end state you want. Remind can be Tertiary (quiet nudge) or Secondary (more assertive). Pick one; don’t show both in production. Show borrower name with the same person icon and positioning you use on Borrowed to minimize cognitive load.
Optional add‑ons for Lent cards:
  • Last reminded timestamp in the meta (e.g., “Reminded 3 days ago”).
  • Contact affordance (e.g., small message icon) if messaging is integrated.
  • Status chip after marking returned (e.g., “Returned on 25 Nov 2025”, neutral chip).
  • If items can be partially returned (sets), consider a mini menu instead of extra buttons.
